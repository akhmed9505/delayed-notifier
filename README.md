# DelayedNotifier

`DelayedNotifier` — это сервис отложенных уведомлений, позволяющий создавать сообщения с заданным временем отправки и доставлять их через различные каналы (Telegram, Email). Уведомления обрабатываются асинхронно через очередь сообщений с поддержкой повторных попыток при ошибках. Используется для сценариев отложенной доставки сообщений: напоминания, события, триггерные уведомления.

---
## Возможности

- Создание отложенных уведомлений через HTTP API
- Получение статуса уведомления по ID
- Отмена запланированных уведомлений до момента отправки
- Асинхронная обработка через очередь RabbitMQ
- Фоновая доставка уведомлений по расписанию
- Повторные попытки отправки при ошибках (retry с задержкой)
- Кэширование статусов через Redis
- Отправка уведомлений через Telegram и Email
- Простой фронтенд для тестирования API и просмотра статусов

## Установка и запуск

### Установка проекта

```bash
git clone https://github.com/akhmed9505/delayed-notifier.git
cd delayed-notifier
```

---
### Конфигурация

Перед запуском проекта скопируйте файл окружения:

```bash
cp .env.example .env
```

После этого заполните значения в `.env`.

**Примечания:**
- **SMTP данные:** Можно использовать сервисы вроде Mailtrap или любой SMTP-провайдер. Укажите логин, пароль, host и port в `.env`.
- **Telegram Chat ID:** Запустите бота в Telegram, затем откройте: `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`. В ответе найдите поле `chat.id` и добавьте его в `.env`.

---
### Запуск проекта

**Запуск через Docker:** 

```makefile
make docker-up
```

После запуска будут доступны сервисы:
- **Backend:** http://localhost:8080
- **Frontend:** http://localhost:3000

Контейнеры поднимают всю инфраструктуру:
PostgreSQL, Redis, RabbitMQ, worker и HTTP API.

**Остановка проекта:**

```makefile
make docker-down
```

Останавливает все сервисы и удаляет `volumes`.

## API endpoints

Доступные эндпоинты:

| Метод | Эндпоинт     | Описание |
|------|--------------|----------|
| POST | /notify      | Создание уведомления |
| GET  | /notify/{id} | Получение статуса |
| DELETE | /notify/{id} | Отмена уведомления |

---
## Примеры запросов

### 1. Создание уведомления

```bash
curl -X POST http://localhost:8080/notify \
  -H "Content-Type: application/json" \
  -d '{
    "channel": "email",
    "recipient": "user@example.com",
    "message": "Test email message",
    "send_at": "2026-04-28T21:15:00+03:00"
  }'
```

**Ответ:**

```json
{
    "id": "d18f9e99-329d-47df-afd6-e871441d0b91"
}
```

---
### 2. Получение статуса уведомления

```bash
curl http://localhost:8080/notify/d18f9e99-329d-47df-afd6-e871441d0b91
```

**Ответ:**

```json
{
  "id": "d18f9e99-329d-47df-afd6-e871441d0b91",
  "status": "pending"
}
```

---
### 3. Отмена уведомления

```bash
curl -X DELETE http://localhost:8080/notify/d18f9e99-329d-47df-afd6-e871441d0b91
```

**Ответ:**

```json
{
    "message": "notification canceled"
}
```

## Frontend

Простой веб-интерфейс доступен по адресу:  
http://localhost:3000

Он предоставляет:
- форму для создания уведомления
- история созданных уведомлений
- возможность отмены запланированного уведомления

Интерфейс используется для тестирования API и содержит только UI-логику отображения и взаимодействия с backend.

## Стек технологий

- Go (Gin framework) — основной язык и HTTP-фреймворк backend
- PostgreSQL — основная реляционная база данных
- Redis — кэширование статусов уведомлений
- RabbitMQ — очередь сообщений для асинхронной обработки уведомлений
- SMTP (Email отправка) — доставка уведомлений по email
- Telegram Bot API — отправка уведомлений в Telegram
- HTML — структура frontend интерфейса
- CSS — стилизация frontend
- JavaScript (vanilla) — логика frontend без фреймворков
- Docker / Docker Compose — контейнеризация и запуск всей системы
- Nginx — раздача frontend и проксирование запросов
- Makefile — команды для сборки и запуска проекта
